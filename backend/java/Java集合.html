# HashMap

HashMap是用来存放键值对的，基于Map接口实现，是非线程安全的

HashMap可以存储null的key和value，但是null作为key只能有一个，作为值可以有多个

HashMap默认初始化容量为16，默认初始化阈值为12，达到阈值需要扩充时，容量会变为原来的2倍

HashMap可能会发生哈希冲突，采用链表法解决，如果数组长度>64,且有链表长度达到阈值,就转化成红黑树，这个阈值默认>=8

# 无参构造+初次put

通常使用HashMap，都是会调用它的无参构造，然后再调用put方法开始放入键值对了，那么，这个过程到底发生了什么？

使用无参构造创建一个HashMap的时候，只是为它的loadFactor设置了一个值（0.75f），loadFactor译为负载系数

之后开始调用put方法，放入键值对，put方法其实内部调用了putVal方法，下面的方法并不完整，只保留了第一次put会执行的部分

参数介绍：

  • hash：通过key计算的hash值
  • key：键值对中的键
  • value：键值对中的值
  • onlyIfAbsent：hashMap中这里是false，表示，put的key相同时，新value会覆盖旧value
  • evict：hashMap这里为true，表示，hashMap不是创建模式
final V putVal(int hash, K key, V value, boolean onlyIfAbsent, boolean evict) {
    	//准备一些变量
        Node<K,V>[] tab; Node<K,V> p; int n, i;
    	//第一次put，tab，table为null，n为0，判断为true
        if ((tab = table) == null || (n = tab.length) == 0)
            n = (tab = resize()).length;//这里调用resize方法，开始设置初始大小
    
    	//这里可以看出，hashMap并不是按照顺序去存放的
        if ((p = tab[i = (n - 1) & hash]) == null)
            tab[i] = newNode(hash, key, value, null);
    
        ++modCount;//修改映射关系或者修改底层结构，都会自增
        if (++size > threshold)//扩容的时候执行
            resize();
        afterNodeInsertion(evict);//什么也没做，一个空方法
        return null;//第一次返回null，说明没有这个键值对映射
    }



final Node<K,V>[] resize() {
        Node<K,V>[] oldTab = table;//刚开始都是null
        int oldCap = (oldTab == null) ? 0 : oldTab.length;// 0
        int oldThr = threshold;
        int newCap, newThr = 0;
        if (oldCap > 0) {
            //无关，略。。。
        }
        else if (oldThr > 0) 
            //无关，略。。。
        else {//会进入这个else               
            newCap = DEFAULT_INITIAL_CAPACITY;// 16
            newThr = (int)(DEFAULT_LOAD_FACTOR * DEFAULT_INITIAL_CAPACITY);// 12
        }
        
        threshold = newThr;// 设置初始阈值为12
        @SuppressWarnings({"rawtypes","unchecked"})
        Node<K,V>[] newTab = (Node<K,V>[])new Node[newCap];//创建一个大小为16的数组
        table = newTab;
        
        return newTab;
}

# HashMap是如何扩容的？

当HashMap的实际大小size>阈值（刚开始是12），会开始扩容，扩容后的容量为之前的2倍，并且新阈值threshold也会变为之前的2倍，扩容其实就是创建了一个2倍容量的新数组，然后遍历原数组，将节点的hash重新与数组长度进行相关运算（hash&(新容量-1)），得到节点在新数组中的下标，然后将节点放到新数组中

final Node<K,V>[] resize() {
        Node<K,V>[] oldTab = table;
        int oldCap = (oldTab == null) ? 0 : oldTab.length;
        int oldThr = threshold;
        int newCap, newThr = 0;
        if (oldCap > 0) {
            if (oldCap >= MAXIMUM_CAPACITY) {
                threshold = Integer.MAX_VALUE;
                return oldTab;
            }
            //扩容关键代码
            else if ((newCap = oldCap << 1) < MAXIMUM_CAPACITY &&
                     oldCap >= DEFAULT_INITIAL_CAPACITY)
                newThr = oldThr << 1; // double threshold
        }
        else if (oldThr > 0) // initial capacity was placed in threshold
            newCap = oldThr;
        else {               // zero initial threshold signifies using defaults
            newCap = DEFAULT_INITIAL_CAPACITY;
            newThr = (int)(DEFAULT_LOAD_FACTOR * DEFAULT_INITIAL_CAPACITY);
        }
        if (newThr == 0) {
            float ft = (float)newCap * loadFactor;
            newThr = (newCap < MAXIMUM_CAPACITY && ft < (float)MAXIMUM_CAPACITY ?
                      (int)ft : Integer.MAX_VALUE);
        }
        threshold = newThr;
        @SuppressWarnings({"rawtypes","unchecked"})
    
    	//创建一个2倍容量的新数组
        Node<K,V>[] newTab = (Node<K,V>[])new Node[newCap];
        table = newTab;
        if (oldTab != null) {
            for (int j = 0; j < oldCap; ++j) {
                Node<K,V> e;
                if ((e = oldTab[j]) != null) {
                    oldTab[j] = null;
                    if (e.next == null)
                        newTab[e.hash & (newCap - 1)] = e;//遍历原数组，将节点放到新数组中
                    
                    else if (e instanceof TreeNode)
                        ((TreeNode<K,V>)e).split(this, newTab, j, oldCap);
                    else {
                        //略。。。
                    }
                }
            }
        }
        return newTab;
    }

# 发生哈希碰撞怎么办？

所谓哈希碰撞，也称为哈希冲突，就是不同的key计算出的hash值相同，由上面的代码得知，如果hash相同，那么计算出的数组中的存放下标就是相同的，于是就发生了，两个键值对，要放在同一个地方的问题

计算出的hash相同有两种情况，一种是，相同的key计算出相同的hash，这种情况，需要用新value覆盖旧value；另一种，不同的key计算出相同的hash，第二种情况也就是哈希碰撞，HashMap解决哈希碰撞的方法是采用链表法

链表法：将具有相同hash的键值对node节点组成一个链表，当需要获取的时候，顺着这个链表查找就行了，如果数组长度>64,且有链表长度达到阈值,就转化成红黑树，这个阈值默认>=8

现在来看看完整的putVal方法

final V putVal(int hash, K key, V value, boolean onlyIfAbsent, boolean evict) {
    
        Node<K,V>[] tab; Node<K,V> p; int n, i;
        if ((tab = table) == null || (n = tab.length) == 0)
            n = (tab = resize()).length;
    
    	//当计算出的hash值不同
        if ((p = tab[i = (n - 1) & hash]) == null)
            tab[i] = newNode(hash, key, value, null);
    
        else {//当计算的hash值相同
            Node<K,V> e; K k;
            
            //如果是key相同，然后计算出相同的hash，则说明只需要覆盖旧值就可以了
            if (p.hash == hash &&
                ((k = p.key) == key || (key != null && key.equals(k))))
                e = p;
            
            //如果是树节点，则执行树处理哈希冲突的方法（刚开始并不是树节点，达到阈值会变成树）
            else if (p instanceof TreeNode)
                e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
            
            //刚开始发生哈希碰撞，链表法进行处理
            else {
                for (int binCount = 0; ; ++binCount) {
                    
                    //结合下面的p = e,移动到链表的末尾，添加新节点
                    if ((e = p.next) == null) {
                        p.next = newNode(hash, key, value, null);
                        
                        //如果数组长度>64,且有链表长度达到阈值,就转化成树，这个阈值默认>=8
                        if (binCount >= TREEIFY_THRESHOLD - 1) 
                            treeifyBin(tab, hash);
                        break;
                    }
                    
                    //遍历链表，看看put的key是不是链表中已经存在的，如果是，就覆盖
                    if (e.hash == hash &&
                        ((k = e.key) == key || (key != null && key.equals(k))))
                        break;
                    
                    p = e;//移动到链表的下一个节点，因为前面e = p.next
                }
            }
            //覆盖旧值
            if (e != null) { // existing mapping for key
                V oldValue = e.value;
                if (!onlyIfAbsent || oldValue == null)
                    e.value = value;
                afterNodeAccess(e);
                return oldValue;
            }
        }
        ++modCount;
    
    	//当实际大小超过threshold，会扩容
        if (++size > threshold)
            resize();
        afterNodeInsertion(evict);
        return null;
    }
